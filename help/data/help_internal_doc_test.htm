关于测试说明文档的说明

测试说明分成下列几部分：
  1. 测试内容简要说明和直接相关的帮助页面
  2. 测试要点和测试项，指出相关的测试案例（只需要页面内链接即可）
  3. 测试案例及案例的简要说明
  4. 其他补充内容，如引用有关的章节信息

测试说明的其他注意事项：

  1. 测试的项目覆盖功能的主要运行逻辑和分支
  2. 测试的预期结果需要写清楚
  3. 测试案例的原始文件，放在ew7/web/help/files/internal_files中，区别于普通帮助中携带的内部文件
  4. 测试案例中附带外部模型文件的，放在ew7/web/help/files/internal_files/model中，建模时使用相对路径
  5. 测试案例中附带的其他外部文件的，放在ew7/web/help/files/internal_files/data中，建模时使用相对路径

系统测试用例命名规范

系统测试用例命名规则为： 软件名-v版本号-功能模块名称-测试子项目名称-ID.后缀名
  • 软件名、版本号、功能模块名、子项目名称等中间断词采用下划线
  • 版本号参考：7_0_0_build_20190114这样的形式，现阶段不必区分build版本号，所以只需要写成...-v7_0_0-...即可
  • 文件夹组织形式：测试类型/软件名_版本号（仅大版本）/功能模块名/子项目名/
  • 内部测试案例放置文件夹：internal_files/test/
    • 系统测试internal_files/test/systest/
    • 单元测试（部分工程文件）internal_files/test/unittest/
    • 集成测试internal_files/test/inttest/
  • 每个子项目增加一个changelog.txt文件，主要用来在测试周期结束前后编写案例内容的变化情况，简要说明案例的变化情况（只需要说明增加、修改案例的主要原因，如果是来自测试系统，应填写有关的测试报告链接），该文件采用简单文本格式和追加方式编写，不得对过去内容进行修改。如果需要归档，则备份并且用备份时间重命名changelog-20190101.txt，保留在原路径下。
  • 本文档不定期对说明内容进行修订
